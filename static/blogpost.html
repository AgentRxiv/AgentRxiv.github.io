Agent Laboratory:

Using LLM Agents as Research Assistants

Samuel Schmidgall

Homepage

Abstract

Recently, major progress has been made toward automating complex tasks that traditionally require a team of humans, including solving repository-level GitHub issues and training machine learning models to solve Kaggle challenges. These systems not only demonstrate high performance, but are also being translated into tools that improve human productivity. However, to date, there is an absence of tools that can assist human researchers similar to code assistants, such as GitHub Copilot and Cursor AI. Toward this, we introduce Agent Laboratory, an open-source large language model (LLM) agent framework for accelerating the individual's ability to perform research, specifically in the machine learning domain. Unlike previous work, where agents produce their own research topics, this work aims to function as a tool to assist human scientists to perform their own research ideas using language agents, rather than automate the entire process. Agent Laboratory takes a human-produced research idea as input and outputs a code repository and a research report. This is accomplished through the use of specialized agents driven by LLMs that collaborate to perform research based on the human's preference for the agent's involvement. In order to accommodate all users, Agent Laboratory is compute flexible, where various levels of resources can be allocated based on the individualâ€™s access to compute (e.g. CPU, GPU, memory) and model inference budget (open-source or closed-source LLM back-ends). We hope Agent Laboratory enables researchers to allocate more effort toward creative ideation and experiment design rather than low-level coding and writing, and ultimately accelerate scientific discovery.

Overview

First and foremost, Agent Laboratory is meant to assist you as the human researcher toward implementing your research ideas. You are the pilot.

Agent Laboratory consists of three primary phases that systematically guide the research process: (1) Literature Review, (2) Experimentation, and (3) Report Writing. During each phase, specialized agents driven by LLMs collaborate to accomplish distinct objectives, integrating external tools like arXiv, Hugging Face, Python, and LaTeX to optimize outcomes. This structured workflow begins with the independent collection and analysis of relevant research papers, progresses through collaborative planning and data preparation, and results in automated experimentation and comprehensive report generation. Details on specific agent roles and their contributions across these phases are discussed in the paper. The modular design ensures compute flexibility, accommodating diverse resource availability while maintaining efficiency in generating high-quality research artifacts.

The presented solver mle-solver aims to act as a general purpose ML code solver. This tool operates by maintaining a collection of top programs which are iteratively modified in order to improve the experimental results according to a scoring function. mle-solver takes as input the following: (1) task instructions, (2) command descriptions, and (3) distilled knowledge. Integrating this information, one of two coding commands are produced by the solver, REPLACE or EDIT. The REPLACE command entirely replaces all of the current code with new code. The EDIT command is usable after the first iteration to make sure code exists, and replaces lines indexed n through m of the current code with as many lines of new code as is produced by the solver. After a code command is executed, the new program is passed through a compiler to check for runtime errors. If it successfully compiles, a score is returned and the list of top programs is updated if the score is higher than the existing programs. If the code does not compile, the agent attempts to repair the code for $N_{rep}$ tries ($N_{rep}$=3 in our experiments) before returning an error and moving on to a new code replacement. In both cases, the agent is encouraged to reflect on its previous command and output to provide as input to the next step.
